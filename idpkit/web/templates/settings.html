API Documentation

Complete reference for the IDP Kit REST API. Also available at /docs (Swagger) and /redoc (ReDoc).

Authentication

All API endpoints require authentication via one of these methods:

  • API Key — Send your key in the X-API-Key header.
  • JWT Bearer Token — Obtain a token from POST /api/auth/login and send it as Authorization: Bearer <token>.
  • Session Cookie — Browsers are automatically authenticated via the session_token cookie set on login.

Rate Limiting

API requests are rate-limited per user. Limits vary by endpoint category and can be configured by admins. When exceeded, the API returns 429 Too Many Requests.

Quick Start Examples

Login and get a JWT token:

curl -X POST /api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "you", "password": "pass"}'

Upload a document (using API key):

curl -X POST /api/documents/ \
  -H "X-API-Key: YOUR_API_KEY" \
  -F "file=@document.pdf"

Query a document:

curl -X POST /api/retrieval/documents/{doc_id}/query \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"query": "What are the key findings?"}'

Execute a Smart Tool:

curl -X POST /api/tools/smart_summary \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"document_id": "DOC_ID", "options": {}}'
